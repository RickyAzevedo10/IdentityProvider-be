Fluxo de registo de utilizador:
1. Endpoint para validar se o username inserido é válido ou não.  
2. Endpoint para definir username validado, password, email e telemóvel, que são os dados necessários para criar o utilizador.

Regras:
- O username é único. Não podem existir usernames repetidos na aplicação.  
- A password deve ter obrigatoriamente 8 caracteres.  
- No momento do login, nunca é pedida a password completa. São sempre pedidas 4 posições aleatórias da password.  
- Por esse motivo, durante a criação do utilizador é necessário gerar e guardar numa tabela todas as combinações possíveis de 4 posições da password de 8 caracteres.  
- A password não deve ser guardada em texto simples. Devem ser usados mecanismos de hash para garantir que o valor real nunca é conhecido.  
- Não é guardada a password completa, apenas os hashes das combinações necessárias para validar o login parcial.

Tarefas:
-Criar as entidades necessárias
-Criar os endpoints adicionais todos dentro de um so controller responsavel por criar users
-Manter a estrutura da clean architecture, e criar camada de serviços de aplicação caso seja necessário
